# liff-request-backend

Backend API รับคำขอเบิกเงินจากฟอร์ม LIFF (ขั้นตอนที่ ①) เก็บลงฐานข้อมูล PostgreSQL
เขียนด้วย Node.js + Express + Prisma

## สิ่งที่ทำได้ตอนนี้

- `POST /requests` — รับข้อมูลจากฟอร์ม LIFF, ออกเลขที่คำขอ, บันทึกลงฐานข้อมูล, **แจ้งเตือน ผบ.หน่วย ทันทีพร้อมปุ่มอนุมัติ/ไม่อนุมัติ**
- `GET /requests` — ดึงรายการคำขอทั้งหมด
- `POST /webhook` — รับ event จาก LINE (ข้อความ + การกดปุ่ม postback) ใช้สำหรับ:
  - ลงทะเบียนผู้อนุมัติ/การเงิน (พิมพ์ข้อความคุยกับ OA)
  - ประมวลผลตอนกดปุ่ม "อนุมัติ" / "ไม่อนุมัติ" แล้วแจ้งผลกลับไปหาผู้ขอเบิก

**ยังไม่ทำ** (ขั้นตอน ③④): หน้าจ่ายเงินของ น.การเงิน, หน้าส่งหลักฐานปิดเรื่อง

**ข้อจำกัดที่ควรรู้**: ไฟล์แนบ (สลิป/ใบเสนอราคา) เก็บไว้ในโฟลเดอร์ `uploads/` บนเครื่อง server ซึ่ง **ไม่ถาวร** — ถ้า Render redeploy หรือ restart เครื่อง ไฟล์จะหายหมด เหมาะสำหรับทดสอบเท่านั้น ก่อนใช้งานจริงควรย้ายไปเก็บที่ Cloudflare R2 (แจ้งได้เมื่อพร้อมทำ)

## 1. รันบนเครื่องก่อน (ทดสอบ)

ต้องมี PostgreSQL สักตัวให้ต่อ — ถ้ายังไม่มีบนเครื่อง ข้ามขั้นนี้ไปสร้างบน Render เลยก็ได้ (ข้อ 3-4) แล้วเอา connection string มาใส่ `.env` เพื่อรันทดสอบบนเครื่องได้เหมือนกัน

```bash
npm install
cp .env.example .env
# แก้ .env ใส่ DATABASE_URL
npx prisma db push
npm run dev
```

เซิร์ฟเวอร์จะรันที่ `http://localhost:3000` — ทดสอบว่าทำงานไหมด้วยการเปิด URL นั้นในเบราว์เซอร์ ควรเห็น `{"ok":true,...}`

## 2. เก็บโค้ดขึ้น GitHub (repo ใหม่แยกจาก frontend)

```bash
git init
git add .
git commit -m "initial backend"
```
สร้าง repo ใหม่บน GitHub (เช่น `liff-request-backend`) แล้ว push ตามที่ GitHub บอก

## 3. สร้างฐานข้อมูลบน Render

1. เข้า [render.com](https://render.com) สมัคร/login (เชื่อมกับ GitHub ได้เลย)
2. Dashboard → **New +** → **PostgreSQL**
3. ตั้งชื่อ เช่น `liff-request-db` → เลือก region ใกล้ๆ (Singapore ถ้ามี) → **Create Database**
4. รอสักครู่ database จะพร้อมใช้งาน → เข้าไปหน้า database นั้น หา **"Internal Database URL"** หรือ **"External Database URL"** — copy เก็บไว้ (จะใช้ External URL ถ้าต่อจากเครื่องเรา หรือ Internal URL ถ้า backend อยู่บน Render เหมือนกัน แนะนำใช้ Internal เพราะเร็วกว่าและฟรีไม่เสียค่า bandwidth)

## 4. สร้าง Web Service บน Render

1. Dashboard → **New +** → **Web Service**
2. เชื่อม GitHub repo `liff-request-backend` ที่เพิ่ง push ไป
3. ตั้งค่า:
   - **Name**: `liff-request-backend`
   - **Region**: เดียวกับ database
   - **Build Command**: `npm install && npx prisma generate`
   - **Start Command**: `npx prisma db push --accept-data-loss --skip-generate && node src/index.js`
   - **Instance Type**: Free ก็ได้สำหรับทดสอบ
4. เลื่อนลงไปที่ **Environment Variables** เพิ่ม:
   - `DATABASE_URL` = Internal Database URL จากข้อ 3
   - `FRONTEND_ORIGIN` = `https://liff-request-form-financial.chateau-huahin.workers.dev`
   - `REQUIRE_LIFF_AUTH` = `false`
5. กด **Create Web Service** รอ build (2-5 นาที)

พอ deploy เสร็จจะได้ URL แบบ:
```
https://liff-request-backend.onrender.com
```
เปิดดูควรเห็น `{"ok":true,...}` เหมือนตอนรันบนเครื่อง

**หมายเหตุ Free tier**: เครื่องจะ sleep เมื่อไม่มีคนเรียกนานเกิน 15 นาที พอมีคนเรียกครั้งแรกจะใช้เวลาปลุกเครื่อง ~30-60 วินาที (ปกติสำหรับ free tier ไม่ใช่ error)

## 5. เชื่อมกลับไปที่ฟอร์ม (Cloudflare)

กลับไปที่ Cloudflare Dashboard → โปรเจกต์ `liff-request-form-financial` → **Settings** → **Build** → **Variables and secrets** → เพิ่ม:
- **Variable name**: `VITE_API_BASE_URL`
- **Value**: `https://liff-request-backend.onrender.com` (URL จากข้อ 4 — **ห้ามมี `/` ปิดท้าย**)

จากนั้น trigger redeploy ฝั่ง frontend (จากเครื่อง ในโฟลเดอร์ `liff-request-form`):
```
git commit --allow-empty -m "connect backend api"
git push
```

## 7. ตั้งค่าขั้นตอน ② — แจ้งเตือน ผบ.หน่วย ให้กดอนุมัติได้

### 7.1 เอา Channel access token + Channel secret มาจาก LINE
1. เข้า [LINE Developers Console](https://developers.line.biz/console/) → เข้า channel **"ขอเบิกเงิน (ธุรการ)"** (ตัว Messaging API ที่สร้างไว้ตั้งแต่แรก ไม่ใช่ตัว LINE Login)
2. แท็บ **Messaging API** → เลื่อนหา **"Channel access token"** → กด **Issue** (ถ้ายังไม่เคยออก) → copy เก็บไว้
3. แท็บ **Basic settings** → เลื่อนหา **"Channel secret"** → copy เก็บไว้

### 7.2 ใส่ค่าใน Render
เข้า Render Dashboard → `liff-request-backend` → เมนูซ้าย **Environment** → เพิ่ม:
- `LINE_CHANNEL_ACCESS_TOKEN` = ค่าที่ copy จากข้อ 7.1
- `LINE_CHANNEL_SECRET` = ค่าที่ copy จากข้อ 7.1
→ Save (Render จะ redeploy ให้อัตโนมัติ)

### 7.3 ตั้ง Webhook URL ที่ LINE
กลับไปที่ channel "ขอเบิกเงิน (ธุรการ)" → แท็บ **Messaging API** → เลื่อนหา **Webhook settings**:
- **Webhook URL**: `https://liff-request-backend.onrender.com/webhook`
- กด **Verify** (ควรขึ้น Success — ถ้า error ให้เช็คว่า Render deploy เสร็จแล้วหรือยัง)
- เปิด **"Use webhook"** เป็น ON

### 7.4 ปิดข้อความอัตโนมัติของ LINE (กันชนกับบอทเรา)
ไปที่ [LINE Official Account Manager](https://manage.line.biz/) → เลือกบัญชี "ขอเบิกเงิน (ธุรการ)" → **การตั้งค่า** → **การตอบกลับอัตโนมัติ**:
- ปิด **"ข้อความตอบกลับอัตโนมัติ"**
- ปิด **"ข้อความต้อนรับ"** (จะปิดหรือเปิดก็ได้ ไม่กระทบระบบ)

### 7.5 ลงทะเบียน ผบ.หน่วย
ให้ ผบ.หน่วย เปิด LINE แล้ว**แอดเพื่อน "ขอเบิกเงิน (ธุรการ)"** เป็นเพื่อนก่อน (หา QR code หรือ Bot basic ID ได้ในแท็บ Messaging API) จากนั้นพิมพ์ข้อความนี้ส่งในแชท:
```
ลงทะเบียนผู้อนุมัติ
```
ถ้าขึ้นตอบกลับ "✅ ลงทะเบียนเป็นผู้อนุมัติ..." แปลว่าสำเร็จ ระบบจดจำ LINE userId ของคนนี้ไว้เป็นผู้อนุมัติแล้ว

### 7.6 ทดสอบทั้งกระบวนการ
1. ผู้ขอเบิกกรอกฟอร์ม LIFF ส่งคำขอ
2. ผบ.หน่วย (คนที่ลงทะเบียนไว้) ควรได้รับข้อความ Flex Message พร้อมรายละเอียดคำขอ + ปุ่ม "✅ อนุมัติ" / "❌ ไม่อนุมัติ" ทันที
3. กดปุ่มใดปุ่มหนึ่ง → ผบ.หน่วย ได้รับข้อความยืนยัน + ผู้ขอเบิกได้รับข้อความแจ้งผลกลับไปที่แชทเดิม

**หมายเหตุ**: ถ้าส่งคำขอไปตอนที่ยังไม่มีใครลงทะเบียนเป็นผู้อนุมัติ คำขอจะถูกบันทึกไว้ปกติ (สถานะ "รออนุมัติ") แต่จะไม่มีการแจ้งเตือนไปหาใคร — ต้องลงทะเบียนผู้อนุมัติก่อนถึงจะเริ่มแจ้งเตือนคำขอใหม่ๆ ที่เข้ามาหลังจากนั้น

## 8. ตั้งค่าขั้นตอน ③ — น.การเงิน จ่ายเงิน

ขั้นตอนนี้ **ใช้ฟอร์ม LIFF ตัวเดิม** (โปรเจกต์ `liff-request-form`) ไม่ต้องสร้าง LIFF app ใหม่ — ฟอร์มจะสลับหน้าอัตโนมัติเมื่อเปิดผ่านลิงก์ที่มี `?requestId=...` ต่อท้าย (ดูรายละเอียดใน README ของ `liff-request-form`)

### 8.1 เพิ่ม env var บน Render
เข้า Render → `liff-request-backend` → **Environment** → เพิ่ม:
- `FRONTEND_LIFF_ID` = ใส่ **ค่าเดียวกับ** `VITE_LIFF_ID` ที่ตั้งไว้ที่ Cloudflare (LIFF ID ที่ได้จาก LINE Developers Console)

### 8.2 ลงทะเบียน น.การเงิน
ให้เจ้าหน้าที่การเงินแอดเพื่อน "ขอเบิกเงิน (ธุรการ)" แล้วพิมพ์ข้อความ:
```
ลงทะเบียนการเงิน
```

### 8.3 ทดสอบ
1. ผู้ขอเบิกส่งคำขอ → ผบ.หน่วย กด "✅ อนุมัติ"
2. น.การเงิน (คนที่ลงทะเบียนไว้) ควรได้รับข้อความพร้อมลิงก์ทันที
3. กดลิงก์ → เปิดฟอร์มจ่ายเงิน เลือกวิธีจ่าย แนบหลักฐาน กดบันทึก
4. ผู้ขอเบิกควรได้รับข้อความแจ้งว่าจ่ายเงินแล้ว พร้อมลิงก์ส่งหลักฐานปิดเรื่อง (ขั้นตอน ④)

## 9. ขั้นตอน ④ — ผู้ขอเบิก ส่งหลักฐานปิดเรื่อง

ไม่ต้องตั้งค่าเพิ่มครับ ใช้ `FRONTEND_LIFF_ID` ตัวเดียวกับขั้นตอน ③ — พอ น.การเงิน กดบันทึกการจ่ายเงินเสร็จ ผู้ขอเบิกจะได้รับลิงก์ส่งหลักฐานอัตโนมัติทันที

### ทดสอบ
1. หลังจ่ายเงินเสร็จ (ทำตามข้อ 8.3) ผู้ขอเบิกจะได้ลิงก์ในข้อความเดียวกับที่แจ้งว่าจ่ายเงินแล้ว
2. กดลิงก์ → เปิดฟอร์มส่งหลักฐาน กรอกยอดใช้จ่ายจริง แนบรูปใบเสร็จ กดส่ง
3. ผู้ขอเบิกได้รับข้อความยืนยันปิดเรื่อง พร้อมสรุปส่วนต่าง (เหลือคืน/ใช้เกิน/พอดี)
4. น.การเงิน ได้รับข้อความสรุปไว้เป็นข้อมูลประกอบบัญชีด้วยเช่นกัน

ถึงตรงนี้ครบทั้ง 4 ขั้นตอนของกระบวนการเบิกเงินแล้วครับ 🎉

## 10. ระบบรายงานสรุปค่าใช้จ่าย

### วิธีเรียกดู — พิมพ์ในแชท
พิมพ์ข้อความนี้คุยกับ "ขอเบิกเงิน (ธุรการ)" (ใครก็พิมพ์ได้ ไม่ได้จำกัดเฉพาะผู้อนุมัติ/การเงิน):
```
รายงานวันนี้
รายงานสัปดาห์นี้
รายงานเดือนนี้
รายงานปีนี้
```
จะได้ Flex Message สรุปยอดเบิกรวม ยอดใช้จริง จำนวนคำขอ แยกตามสถานะและหมวดหมู่ พร้อมปุ่ม "📊 ดูรายงานแบบเต็ม" ที่ลิงก์ไปหน้าเว็บสรุปแบบละเอียด (ต้องตั้ง `FRONTEND_LIFF_ID` ไว้แล้ว ดูข้อ 8.1) และปุ่ม **"📄 ดาวน์โหลด PDF"** ที่โหลดไฟล์ PDF จริงมาส่งต่อ/ปริ้นได้ทันที (ต้องตั้ง `BACKEND_BASE_URL` — ดูข้อ 11)

### Endpoint สำหรับหน้ารายงานแบบเต็ม
`GET /reports?period=daily|weekly|monthly|yearly` — คืนค่า JSON: `label`, `count`, `totalRequested`, `totalActual`, `byCategory`, `byStatus`, `requests` (รายการทั้งหมดในช่วงนั้น)

สัปดาห์เริ่มวันจันทร์ ตามเวลาไทย (UTC+7) ไม่ต้องตั้งค่าอะไรเพิ่ม

## 11. ดาวน์โหลดรายงานเป็น PDF

### ตั้งค่า
เพิ่ม env var บน Render:
- `BACKEND_BASE_URL` = URL ของ backend ตัวเอง เช่น `https://liff-request-backend.onrender.com` (**ห้ามมี `/` ปิดท้าย**)

### วิธีใช้
- กดปุ่ม "📄 ดาวน์โหลด PDF" จากข้อความสรุปรายงานในแชท — ได้ไฟล์ PDF ทันที ไม่ต้องผ่านฟอร์ม LIFF หรือเบราว์เซอร์ภายนอกเลย
- หรือเปิด `GET {BACKEND_BASE_URL}/reports/pdf?period=daily|weekly|monthly|yearly` ตรงๆ

PDF มีฟอนต์ไทย (Noto Sans Thai) ฝังในไฟล์อยู่แล้วที่ `assets/fonts/NotoSansThai.ttf` — เปิด/แชร์/ปริ้นได้จากทุกอุปกรณ์โดยไม่ต้องพึ่งฟังก์ชันพิมพ์ของเบราว์เซอร์ (ซึ่งบางเครื่อง/เบราว์เซอร์ทำงานไม่เสถียรเมื่อเปิดผ่าน LINE)

**สำคัญ — วิธีแชร์ไฟล์ PDF ต่อให้ถูกต้อง**: หลังกดลิงก์แล้วไฟล์จะถูกดาวน์โหลดเก็บไว้ในเครื่อง (โฟลเดอร์ Downloads/ไฟล์) ต้อง**เข้าไปแชร์จากแอปจัดการไฟล์ (Files/My Files) หรือแตะที่การแจ้งเตือนดาวน์โหลดโดยตรง** แล้วเลือกแชร์ไปที่ LINE จากตรงนั้น — ถ้ากดแชร์จาก "แท็บเบราว์เซอร์ที่เปิด PDF ค้างอยู่" บางเครื่อง/เบราว์เซอร์จะแชร์ "ลิงก์เว็บ" แทนไฟล์จริง ทำให้ฝั่งผู้รับเห็นเป็นข้อความลิงก์ ไม่ใช่ไฟล์ PDF

## 12. ส่งรูปภาพสรุปเข้าแชทอัตโนมัติ (ไม่ต้องกดอะไรเพิ่ม)

เพิ่มมาแก้ปัญหาข้อจำกัดของ LINE ที่ **ไม่รองรับการส่งไฟล์ PDF ผ่าน Messaging API โดยตรง** (LINE อนุญาตแค่ text/รูปภาพ/Flex Message เท่านั้น) — ตอนนี้พอพิมพ์คำสั่ง "รายงานวันนี้" ฯลฯ ระบบจะแนบ**รูปภาพสรุป**ต่อท้าย Flex Message ให้อัตโนมัติทันที ไม่ต้องกดลิงก์ใดๆ เลย

- ใช้ `BACKEND_BASE_URL` ตัวเดียวกับข้อ 11 (ถ้ายังไม่ตั้งจะข้ามการส่งรูปไปเงียบๆ)
- Endpoint ที่ใช้: `GET /reports/image?period=daily|weekly|monthly|yearly` คืนค่าเป็น PNG
- รูปภาพเป็นสรุปแบบย่อ (ยอดรวม, แยกสถานะ, แยกหมวดหมู่) — รายละเอียดเต็มทุกรายการยังต้องดูจากปุ่ม "ดูรายงานแบบเต็ม" หรือ PDF
